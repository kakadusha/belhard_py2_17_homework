типы тэгов
    строчные -  <img>, <span>, <a>, <q>, <code>
        Внутрь строчных элементов допустимо помещать текст или другие строчные элементы. 
        Вставлять блочные элементы внутрь строчных запрещено.
        Эффект схлопывания отступов не действует.
        Свойства, связанные с размерами (width, height) не применимы.
        Ширина равна содержимому плюс значения отступов, полей и границ.
        Несколько строчных элементов идущих подряд располагаются на одной строке 
                и переносятся на другую строку при необходимости.
        Можно выравнивать по вертикали с помощью свойства vertical-align
   блочные - <address>, <blockquote>, <div>, <fieldset>, <form>, <h1>,...,<h6>, 
                <hr>, <ol>, <p>, <pre>, <table>, <ul>
        Блоки располагаются по вертикали друг под другом.
        По ширине блочные элементы занимают всё допустимое пространство.        
        Высота блочного элемента вычисляется браузером автоматически, исходя из содержимого блока.
        На блочные элементы не действуют свойства, предназначенные для строчных элементов, вроде vertical-align.
        Текст по умолчанию выравнивается по левому краю.





CЕЛЕКТОРЫ
    p - через тэг
    .class - через класс
    #id  - через id

    p, a - все р и а
    div p - все потомки
    .class p
    div>p - только один потомок
    div ul + ul - плюс следующий элемент после указанного 
    br ~ p - все р перед которыми есть br
    input[type='text']
    a[target='_blank'] - только конкретные а
    a[href^='https'] - url начинается на https
    a[href$='.by'] - url заканчивается на .by
    a[href*='task'] - url содержит task
    :nth-child(2n) - каждый второй

РАЗМЕРЫ
    - px
    - % - от родительского элемента
    - vw vh - процент от экрана (viewport)
    - em - процент от базового шрифта родительского элемента
    - rem - процент от базового шрифта базового элемента
    - cm, mm, in - абсолютные величины - редкие



Позиционирование
static - по дефолту
relative - смещение top left относительно родной позиции при этом родное место не заполняется
absolute -  Элемент исчезает с того места, где он должен быть и позиционируется заново. 
                Остальные элементы, располагаются так, как будто этого элемента никогда не было. 
                Координаты top/bottom/left/right для нового местоположения отсчитываются от 
                ближайшего позиционированного родителя, т.е. родителя с позиционированием, 
                отличным от static. Если такого родителя нет – то относительно документа.
fixed - всегда относительно окна браузера. Ширина элемента устанавливается по содержимому.
sticky - по окну браузера но при скроле залипает в пределах родителя
