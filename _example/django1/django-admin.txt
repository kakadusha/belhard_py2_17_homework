check - для проверки всего проекта Django на наличие общих проблем.
	django-admin check auth admin myapp

compilemessages - для создания файлов перевода с указанием локалей


dbshell - Запускает клиент командной строки для движка базы данных

diffsettings - Отображает различия между текущим файлом настроек и настройками Django по умолчанию (или другим файлом настроек, указанным через --default

dumpdata - Выводит на стандартный вывод все данные в базе данных
	--output OUTPUT

flush - Удаляет все данные из базы данных

inspectdb - Интроспекция таблиц базы данных в базе данных

loaddata - Ищет и загружает в базу данных содержимое именованного fixture(dump базы)

makemessages - Просматривает все дерево исходных текстов текущего каталога и извлекает все строки, помеченные для перевода

makemigrations - Создает новые миграции на основе изменений, обнаруженных в ваших моделях

migrate - Синхронизирует состояние базы данных с текущим набором моделей и миграций

runserver

sendtestemail - Отправляет тестовое письмо

shell - если пайтонов несколько указывает какой запускать 
	django-admin shell -i ipython

showmigrations	- Показывает все миграции в проекте. Вы можете выбрать один из двух форматов

sqlflush - ыводит операторы SQL, которые будут выполнены для команды flush

sqlmigrate - Печатает SQL для именованной миграции.

sqlsequencereset - Выводит SQL-запросы для сброса последовательностей для заданного имени (имен) приложения.

squashmigrations - Сжимает миграции для app_label до migration_name включительно в меньшее количество миграций, если это возможно

startapp - Создает структуру каталога приложения Django для заданного имени приложения в текущем каталоге или в заданном месте назначения.

startproject - создает структуру 

test - Запускает тесты для всех установленных приложений

testserver - Запускает сервер разработки Django (как в runserver), используя данные из заданного файла
